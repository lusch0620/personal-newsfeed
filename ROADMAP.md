## Architecture (current)

```
GitHub Actions (cron, every ~2h)
  -> fetch_feeds.py       config/sources.yaml -> data/raw_articles.json
  -> fetch_notes.py       Gist -> data/notes.md (injected into scoring)
  -> summarize.py         Haiku x8 async workers -> data/feeds.json
  -> generate_briefs.py   Haiku per lane + Yahoo Finance -> data/briefs.json
  -> copies both to docs/data/
  -> commits + push -> GitHub Pages serves docs/

Browser
  -> reads docs/data/feeds.json + docs/data/briefs.json
  -> Ask Claude: POST newsfeed-ai.luciusgao2001.workers.dev/ (Haiku)
  -> Brief Me:   POST newsfeed-ai.luciusgao2001.workers.dev/brief (Sonnet + web search)
  -> notes/signals -> Gist sync -> next pipeline run picks them up
```

## Possible next phases

These were not built but are natural extensions. Pick up here in a future session.

### ~~PWA / mobile install~~ ✅ 2026-06-29
`docs/manifest.json` + `docs/sw.js` + iOS meta tags in `index.html`.
- Home screen install on iPhone (Safari → Add to Home Screen) and Android (Chrome install prompt)
- Full-screen standalone launch, dark status bar
- Offline: network-first for `feeds.json` / `briefs.json`, cache fallback serves stale data
- Icons: `docs/icons/icon-192.png` and `icon-512.png` generated by `scripts/generate_icons.py`

### Cross-edition deduplication + article archive
Articles read yesterday are re-appearing today because read state is keyed to the current edition date, not the article ID. Two parts:
- **Dedup:** track read articles by ID across multiple days so they don't resurface
- **Archive browser:** let user pull up previous days' editions and search/reference past articles
- Effort: ~3-4 hours. Requires rethinking the edition key and adding an archive data structure.

### Daily email digest
Pipeline sends a morning email with top 5 stories + lane brief so you don't have to open the site.
- Add a `send_digest.py` script that runs on a separate cron (e.g. 7am ET weekdays only)
- Use a free transactional email service: **Resend** (100 emails/day free) or **SendGrid** (100/day free)
- Email template: lane brief lede + top 5 articles with digest + source link
- Store `RESEND_API_KEY` (or equivalent) as a GitHub Actions secret
- Effort: ~3-4 hours. Requires signing up for Resend/SendGrid.

### Source management UI
Add/remove RSS sources from the feed directly in the browser without editing `sources.yaml`.
- Add a "Sources" panel in the site (new mode in the header nav)
- List current sources per lane; allow toggling on/off and adding new RSS URLs
- Changes written back to Gist as a `sources-overrides.json` file
- Pipeline reads overrides and merges with `sources.yaml` before fetching
- Effort: ~1 day. Mostly frontend work.

### Full-text search
Search across today's edition and (optionally) archived articles.
- Add a search input that filters `DATA` client-side by title, summary, topics, source
- For archive search: store past `feeds.json` snapshots in Gist or a small R2 bucket
- Effort: client-side only search is ~2 hours; archive search adds complexity.

## Risks / open decisions

- **Brief Me cost:** Sonnet + web search is more expensive than Haiku. Each Brief Me call costs ~$0.01-0.03. Fine for personal use; revisit if usage spikes.
- **Yahoo Finance:** Unofficial API, no auth required. Could break without notice. Fallback to demo data is already wired.
- **Some feeds rate-limit or change RSS paths.** Audit `sources.yaml` quarterly.
- **Citywire RIA + Business Wire** still disabled in `scrape_targets` — JS-rendered, need Playwright to scrape properly.
- **WSJ feeds** are headline+blurb only (paywalled). Reader "open source" link still works.
